Adicionando arquivos no Git

Após a criação de um arquivo ou arquivos, ele(s) ainda não estão sendo monitorados. Aqui estão algumas formas de adicioná-los.

Para adicionar um arquivo de cada vez, digite:

$ git add arquivo.extensao $ git add arquivo2.extensao

Ou para adicionar mais de um ao mesmo tempo:

$ git add arquivo.extensao arquivo2.extensao

Caso queira adicionar todos os arquivos de um diretório recursivamente, o que inclui seus subdiretórios e arquivos ou pastas ocultas:

$ git add .

Removendo arquivos do index

Remoção antes do primeiro commit

Vamos supor que adicionamos um terceiro arquivo sem querer (arquivo3.extensao). Não era o que queríamos. Então devemos tirá-lo do index. O index é como uma "sala de espera" onde ficam os arquivos que serão "commitados".

$ git rm --cached arquivo3.extensao
  • Esse código é apenas para o caso de não ter sido feito nenhum commit ainda.

    • Para retirar todos os arquivos, podemos usar o comando:

    $ git rm --cached -r *

    Remoção após o primeiro commit

    Caso não seja o primeiro commit e você precise excluir um arquivo do index, utilize o comando:

    $ git reset HEAD arquivo.extensao

    Para retirar do index todos os arquivos, digite:

    $ git reset HEAD .
    Enviando arquivos para o repositório Git

    Vamos supor que temos no nosso repositório a seguinte situação: temos dois arquivos novos e vamos fazer o nosso primeiro commit.

    Ao colocar os arquivos dentro do repositório, precisamos informar ao Git o motivo desses arquivos, com o comando:

    $ git commit -m 'escreva sua mensagem aqui'
  • pode-se usar aspas simples ou duplas para escrever a mensagem.
    • Corrigindo o último commit

    Vamos supor que adicionei dois arquivos com os seguintes commits:

    $ git add index.html estilo.css
    $ git commit -m 'adicionando arquivo html e css'

    E em seguida, adiciono mais outro, mas com o commit errado:

    $ git add script.js
    $ git commit -m 'script python'

    Vamos modificar o último commit com o seguinte comando:

    $ git commit --amend -m 'adicionando script js'
    Ignorando arquivos no Git

    Caso você queira que o git não enxergue determinados arquivos dentro do seu diretório que está sendo rastreado, precisamos criar o seguinte arquivo:

    $ touch .gitignore
  • o comando touch serve para criar arquivos no Linux pelo terminal.
  • .gitignore deve ser o nome do nosso arquivo de texto.

    • Abrindo esse arquivo, vamos digitar dentro dele o que queremos deixar de fora do repositório. É importante notar que devemos informar um tipo de arquivo por linha.

    Os arquivos descritos dentro do arquivo .gitignore não aparecem no Working directory.

  • Working directory: é onde estão seus arquivos que serão trabalhados. Onde ficam os arquivos que ainda não receberam o comando commit.

    • Agora basta adicionar o arquivo .gitignore e fazer o commit dele.
